**Пример:** Нужно решить, как показывать прогресс анализа контракта в реальном времени в UI. Сейчас фронтенд опрашивает API каждые несколько секунд, что увеличивает нагрузку и задерживает отклик. Требуется решение, которое уменьшит опросы, покажет прогресс в течение секунды после обновлений на бэкенде и будет работать с текущим Contractor API и Reverb.

## Решение

Сформулируйте решение чётко и конкретно.

**Пример:** Будем использовать Laravel Echo (протокол Pusher) для подписки на канал по контракту после старта анализа. Бэкенд будет рассылать события прогресса; фронтенд в `lib/echo.ts` подпишется, будет обновлять локальное состояние по каждому событию и отписываться при завершении анализа или уходе пользователя со страницы. Сохраняем резервный вариант: при отсутствии конфигурации Echo или сбое подключения — возврат к текущему опросу.
