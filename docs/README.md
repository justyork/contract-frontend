## Стандарты документации

1. **Детерминированность:** инструкции однозначны и воспроизводимы.
2. **Структурированность:** единообразные заголовки, таблицы и блоки кода.
3. **Читаемость для AI:** чёткие разделы и конкретные примеры.
4. **Конкретность:** без разделов-заглушек и только TODO; реальные примеры и правила.
5. **Декларативность:** настоящее время; описывать, что есть, а не что изменилось.

## Обновление документации

При обновлении документации:

1. Сохранять структуру выше; новые файлы добавлять в соответствующий раздел.
2. Добавлять конкретные примеры (код, конфиг или поток).
3. При необходимости обновлять перекрёстные ссылки и ссылки в AGENTS.md.
4. Использовать таблицы для соответствий и чек-листов где уместно.
